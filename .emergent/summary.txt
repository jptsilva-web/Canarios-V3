<analysis>**original_problem_statement:**
The user wants to create a Canary Breeding Management Application, similar to a system shown in a YouTube video (Orniplus). The application should run locally and manage the entire breeding process.

**PRODUCT REQUIREMENTS:**
- **Dashboard**: A central view with key statistics, daily tasks, and recent activity.
- **Zone & Cage Management**: Visually create and manage zones with a grid of cages. Cages are color-coded based on breeding status.
- **Bird Registry**: A database of all birds with unique band numbers, year, gender, STAM, and Class.
- **Pair Management**: Create breeding pairs by assigning birds to a cage.
- **Clutch & Egg Tracking**: Track clutches and the status of individual eggs (laid, fertile, infertile, hatched). Hatched eggs should show the band number.
- **Newborns Page**: A list of newly banded chicks with their details.
- **Task Management**: A calendar for automatically generated and manually added tasks.
- **Email Notifications**: All tasks should trigger an email notification.
- **Settings Page**: Configure breeding cycle timings and email settings.
- **Genealogy**: A family tree visualization for each bird.
- **Statistics & Reports**: A page with detailed breeding statistics, performance trends, and year-over-year comparisons.
- **Data Export**: Ability to export data to CSV/PDF.
- **Multi-language Support**: The application should be translatable into Portuguese, English, and Spanish, with a visual flag selector.
- **Season Management**: The ability to manage data by breeding season/year.
- **Printable Breeding Cards**: Generate printable cards for each breeding pair.
- **IIS Compatibility**: User wants to be able to deploy the application on a Windows server using IIS.
